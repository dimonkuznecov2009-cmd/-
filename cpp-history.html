📖 Рождение легенды: C with Classes

Бьярне Страуструп (Bjarne Stroustrup) — датский программист, работавший в легендарной Bell Labs (AT&T). В конце 1970-х годов он работал над задачами моделирования распределённых систем и был недоволен доступными инструментами.

Язык C был быстрым и низкоуровневым, но не имел механизмов объектно-ориентированного программирования. Язык Simula имел классы, но был слишком медленным. Страуструп решил создать нечто среднее.

В 1979 году он начал работу над "C with Classes" ("Си с классами"). Первая версия добавляла к C: классы, наследование, строгую типизацию, встраиваемые функции (inline) и проверку аргументов по умолчанию.

💡 Интересный факт: Изначально язык назывался "C with Classes". В 1983 году его переименовали в C++. Знак "++" — оператор инкремента в языке C, символизирующий эволюционное развитие: "C плюс один шаг вперёд".

«Я хотел создать язык, который сочетал бы эффективность и гибкость C с выразительностью и абстракциями Simula».

— Бьярне Страуструп

«C++ был спроектирован так, чтобы дать программисту выбор: платить за то, что он использует, и не платить за то, что не использует. Никаких скрытых накладных расходов».

📦 Первые шаги: 1985 год

C++ — первое издание

В 1985 году вышла первая коммерческая версия C++. В том же году Бьярне Страуструп опубликовал легендарную книгу "The C++ Programming Language" ("Язык программирования C++"), которая стала настольной для целого поколения разработчиков.

Ключевые возможности первого релиза:

  • Классы и объекты
  • Наследование (одиночное)
  • Виртуальные функции (позднее связывание)
  • Дружественные функции (friend)
  • Перегрузка функций
  • Операторы new и delete
// Первый C++ код
#include 

class Hello {
public:
    void say() {
        std::cout << "Hello, C++!" << std::endl;
    }
};

int main() {
    Hello h;
    h.say();
    return 0;
}

📅 Полная хронология развития C++

1979-1983
C with Classes
🔬 Зарождение: "Си с классами"

Бьярне Страуструп в Bell Labs начинает разработку. Добавлены классы, наследование, inline-функции. В 1983 году язык переименован в C++.

  • Классы и объекты
  • Наследование (одиночное)
  • Виртуальные функции
1985
C++ 1.0
📚 Первая коммерческая версия

Выход первой официальной версии и книги "Язык программирования C++". Язык начинает набирать популярность в промышленной разработке.

  • Первая коммерческая версия
  • Книга Бьярне Страуструпа
  • Выход iostream (потоковый ввод/вывод)
1989
C++ 2.0
🧬 Множественное наследование

Добавлено множественное наследование (класс может наследовать от нескольких родителей), статические функции-члены, защищённые члены.

  • Множественное наследование
  • Статические члены класса
  • Защищённые (protected) члены
1990
The Annotated C++
📖 Шаблоны (Templates)

Выходит книга "The Annotated C++ Reference Manual" (ARM), которая закладывает основы для стандартизации. Появляются шаблоны (templates) — основа обобщённого программирования.

  • Шаблоны (templates) — обобщённое программирование
  • Обработка исключений (exceptions)
  • Пространства имён (namespaces)
1998
C++98 (ISO/IEC 14882:1998)
🌟 Первый международный стандарт

Важнейший момент в истории! C++ становится международным стандартом ISO. Добавлена STL (Standard Template Library) — стандартная библиотека шаблонов, созданная Александром Степановым.

  • STL (контейнеры, алгоритмы, итераторы)
  • RTTI (информация о типах во время выполнения)
  • dynamic_cast и typeid
  • Библиотека iostream стандартизирована
💡 Значение: C++98 стал фундаментом, на котором строится современный C++. Многие компании до сих пор используют этот стандарт для легаси-кода.
2003
C++03
🔧 Исправление ошибок

Небольшое обновление стандарта C++98. Исправлены ошибки, улучшена совместимость. Фактически — C++98 с патчами.

  • Исправление ошибок стандарта
  • Улучшенная совместимость
  • Без новых фич
2011
C++11 (Modern C++)
⚡ Революция: Modern C++

Самый важный релиз после C++98! После 13 лет затишья комитет стандартизации выпустил масштабное обновление. C++11 сделал язык современным, удобным и безопасным.

  • Лямбда-выражения [](){}
  • Умные указатели (shared_ptr, unique_ptr, weak_ptr)
  • Семантика перемещения (move semantics)
  • auto — автоматическое выведение типа
  • decltype — получение типа выражения
  • for (auto x : container) — range-based for
  • nullptr — безопасный нулевой указатель
  • constexpr — вычисления на этапе компиляции
  • Статическая проверка (static_assert)
  • Многопоточность (std::thread, std::mutex)
  • Библиотека хроно (std::chrono)
  • Вариативные шаблоны (...)
  • Списки инициализации
💡 Значение: Именно C++11 называют "Modern C++". Старый стиль кода (сырые указатели, new/delete) ушёл в прошлое. Умные указатели и move-семантика изменили подход к управлению ресурсами.
2014
C++14
🔧 Улучшения и доработки

Небольшое обновление, расширяющее возможности C++11. Улучшены лямбды и constexpr.

  • Generic lambdas (auto в параметрах)
  • Переменные шаблоны
  • Расширенный constexpr
  • Библиотека make_unique
2017
C++17
📦 Новые возможности

C++17 принёс множество полезных функций, упрощающих повседневное программирование. Файловые системы, параллельные алгоритмы, структурированные привязки.

  • std::optional — необязательные значения
  • std::variant — типобезопасное объединение
  • std::any — хранение любого типа
  • std::filesystem — работа с файловой системой
  • Структурированные привязки auto [a,b] = ...
  • if constexpr — условная компиляция
  • Параллельные алгоритмы STL
  • inline переменные
2020
C++20
🚀 Крупнейшее обновление после C++11

Огромный релиз, добавляющий концепты, модули, корутины и ranges. Язык становится ещё мощнее и удобнее.

  • Концепты (concepts) — требования к типам шаблонов
  • Модули (modules) — ускорение компиляции, замена #include
  • Корутины (coroutines) — асинхронное программирование
  • Ranges библиотека — работа с последовательностями
  • std::span — немедленно владеющий вид на массив
  • std::format — как Python f-строки
  • Трёхстороннее сравнение (<=> operator)
  • constexpr для std::vector и std::string
💡 Значение: Концепты наконец-то позволяют писать понятные сообщения об ошибках для шаблонов. Модули могут ускорить компиляцию в 10-20 раз. Это самый значительный релиз за последние 10 лет.
2023
C++23
🔧 Эволюция, а не революция

C++23 — небольшое обновление, дополняющее возможности C++20. Улучшена стандартная библиотека.

  • std::expected — обработка ошибок без исключений
  • std::mdspan — многомерные массивы
  • std::print — упрощённый вывод (как Python print)
  • this в лямбдах ([=, this])
  • std::string::contains() — проверка подстроки
2026 (планируется)
C++26
🔮 Будущее C++

Ожидаемые нововведения в следующем стандарте.

  • Рефлексия (compile-time reflection)
  • Улучшенная поддержка сетей (networking TS)
  • std::execution — новый подход к параллелизму
  • Улучшения для корутин
  • Расширение возможностей constexpr

⚡ Ключевые особенности C++

⚙️

Низкоуровневое управление памятью

Полный контроль над памятью через new/delete, умные указатели, размещение в куче и стеке. Нет скрытых накладных расходов.

🚀

Высочайшая производительность

Компилируется напрямую в машинный код. Оптимизации на этапе компиляции. Равен по скорости языку C, но с более высокоуровневыми абстракциями.

🧬

Многопарадигмальность

Процедурное, ООП, обобщённое (шаблоны), функциональное (лямбды), метапрограммирование. Выбирайте стиль под задачу.

📝

Статическая типизация

Ошибки типов отлавливаются на этапе компиляции. Строгая проверка типов. Вывод типов (auto) облегчает жизнь.

🔧

RAII (Resource Acquisition Is Initialization)

Главная идиома управления ресурсами. Ресурс захватывается в конструкторе, освобождается в деструкторе. Безопасно и автоматически.

🧵

Встроенная многопоточность

std::thread, std::async, std::mutex, атомарные операции. Полноценная поддержка параллелизма и конкурентности.

📦

Шаблоны (Templates)

Обобщённое программирование. Код, работающий с любыми типами. Мощное метапрограммирование. Концепты (C++20) для лучших сообщений об ошибках.

🔁

Обратная совместимость с C

Практически любой C-код компилируется как C++. Можно использовать C-библиотеки напрямую.

🛡️ Умные указатели: как C++ решает проблемы с памятью

Unique_ptr, Shared_ptr, Weak_ptr — безопасное управление ресурсами

std::unique_ptr
Эксклюзивное владение. Нельзя скопировать, только переместить. Замена сырым указателям.
auto ptr = std::make_unique<int>(42);
std::shared_ptr
Разделяемое владение. Счётчик ссылок. Удаляется, когда счётчик == 0.
auto ptr = std::make_shared<int>(42);
std::weak_ptr
Слабая ссылка на shared_ptr. Решает проблему циклических ссылок.
std::weak_ptr<int> weak = shared;

🔒 RAII — главная идиома C++

Resource Acquisition Is Initialization — получение ресурса есть инициализация. Ресурс (память, файл, мьютекс) захватывается в конструкторе, освобождается в деструкторе. Это гарантирует отсутствие утечек даже при исключениях.

class FileHandle {
    FILE* f;
public:
    FileHandle(const char* name) : f(fopen(name, "r")) {}
    ~FileHandle() { if(f) fclose(f); } // Автоматическое закрытие!
    // Запрет копирования, разрешение перемещения...
};
