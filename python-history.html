🎄 Рождественский эксперимент 1989 года

Гвидо ван Россум — нидерландский программист, работавший в CWI (Centrum Wiskunde & Informatica) в Амстердаме. В декабре 1989 года, во время рождественских каникул, он искал интересный проект, чтобы занять себя. Так родилась идея создать преемника языка ABC.

Язык ABC, над которым Гвидо работал ранее, был простым и понятным, но страдал от отсутствия обработки исключений, плохой модульности и невозможности работать с сетью. Гвидо решил исправить эти недостатки, сохранив при этом простоту и читаемость.

Почему Python? Название не имеет ничего общего со змеями. Гвидо был поклонником британского комедийного шоу «Летающий цирк Монти Пайтона» (Monty Python's Flying Circus). Он хотел, чтобы название было коротким, запоминающимся и немного загадочным.

«В рождественскую неделю у меня было много времени, и я решил написать язык, который бы вдохновлялся ABC, но был бы более амбициозным».

— Гвидо ван Россум

«Моей главной целью было создать язык, который был бы одновременно мощным и приятным в использовании. Я хотел, чтобы код на Python читался как псевдокод».

📅 Полная хронология развития Python

Февраль 1991
Python 0.9.0
🐍 Первый публичный релиз

Python выходит в мир. Уже тогда поддерживал классы, исключения, функции и модули. Сообщество начинает формироваться вокруг новости alt.sources.

  • Ключевая фича: модульная система
  • Объём кода: ~10 000 строк на C
Январь 1994
Python 1.0
📊 Функциональное программирование

Добавлены map, filter, reduce и lambda-выражения из мира Lisp. Появляется первая версия стандартной библиотеки, включая модули для работы с регулярными выражениями.

  • lambda, map, filter, reduce
  • Первый инструмент для создания GUI — Tkinter
  • Python начинает набирать популярность в научных кругах
Октябрь 2000
Python 2.0
♻️ Эпоха Python 2.x

Ключевое обновление! Добавлен сборщик мусора для циклических ссылок, унификация типов и классов. Python 2.x становится стандартом на следующие 8 лет. Именно эту версию изучали миллионы программистов начала 2000-х.

  • Сборщик мусора (циклические ссылки)
  • Генераторы списков (list comprehensions)
  • Поддержка Unicode (частичная)
Декабрь 2008
Python 3.0
⚠️ Революция: Python 3 без обратной совместимости

Самый спорный и важный релиз в истории Python. Разработчики решили исправить фундаментальные недостатки языка, даже если это сломает совместимость с Python 2.

  • print стал функцией: print("Hello") вместо print "Hello"
  • Все строки стали Unicode по умолчанию
  • Изменено деление целых чисел: 5/2 = 2.5 (а не 2)
  • Добавлены нелокальные переменные (nonlocal)
⚠️ Переход был настолько болезненным, что сообщество жило с двумя ветками более 10 лет. Официальная поддержка Python 2 прекращена только 1 января 2020 года.
Сентябрь 2015
Python 3.5
⏳ Асинхронность и матрицы

Введение async/await сделало Python конкурентоспособным в асинхронном программировании. Оператор @ открыл новую эру для научных вычислений.

  • async/await — асинхронное программирование
  • Оператор @ для умножения матриц
  • Типизированные словари (PEP 484)
Декабрь 2016
Python 3.6
📝 f-строки — революция форматирования

f-строки (f"Hello {name}") стали самым удобным способом форматирования строк. Аннотации переменных позволили писать более типобезопасный код.

  • f-строки — самый удобный форматировщик
  • Аннотации переменных
  • Асинхронные генераторы
Октябрь 2019
Python 3.8
walrus operator (:=)

Моржовый оператор := позволяет присваивать и использовать значение в одном выражении. Код стал короче, но споры о читаемости не утихают.

  • Моржовый оператор :=
  • Позиционные параметры (/)
  • f-строки с = для отладки
Октябрь 2021
Python 3.10
pattern matching (match/case)

Python получил структурный pattern matching, похожий на switch в других языках, но гораздо мощнее. Улучшенные сообщения об ошибках помогают новичкам.

  • match/case — структурный pattern matching
  • Улучшенные сообщения об ошибках
  • Родительские контекстные менеджеры
Октябрь 2023
Python 3.12
⚡ Производительность и новый синтаксис

Значительное ускорение интерпретатора (до 10-20% в некоторых задачах). Улучшенная поддержка generic-классов и f-строк.

  • Ускорение интерпретатора на 5-20%
  • Улучшенные f-строки (без ограничений)
  • Новый синтаксис generic-классов
Октябрь 2024
Python 3.13
🚀 Экспериментальный JIT и no-GIL

Самый амбициозный релиз за последние годы. Экспериментальный JIT-компилятор ускоряет горячий код в 2-5 раз. Опциональный режим без GIL (PEP 703) позволяет использовать все ядра процессора.

  • Экспериментальный JIT-компилятор
  • Опциональный режим без GIL (free-threaded)
  • Улучшенная цветная трассировка ошибок

⚡ Ключевые особенности Python

📖

Читаемость кода

Отступы вместо фигурных скобок. Код на Python читается как псевдокод — это главная причина, почему его выбирают для обучения.

🧩

Динамическая типизация

Переменные не требуют объявления типа. Пишите быстрее, но будьте внимательнее — ошибки проявляются только во время выполнения.

♻️

Автоматическое управление памятью

Сборщик мусора и подсчёт ссылок. Забудьте про malloc и free — Python всё делает сам.

📦

Огромная экосистема

PyPI содержит более 400 000 библиотек. pip install всё, что нужно — от нейросетей до парсинга PDF.

🧵

Мультипарадигмальность

ООП, процедурное, функциональное — выберите стиль, который подходит для вашей задачи.

🌍

Кроссплатформенность

Windows, macOS, Linux, даже Android и iOS. Один код — везде работает.

🧘 Дзен Python (The Zen of Python)

>>> import this

Красивое лучше уродливого.

Явное лучше неявного.

Простое лучше сложного.

Сложное лучше запутанного.

Плоское лучше вложенного.

Разреженное лучше плотного.

Читаемость имеет значение.

Особые случаи не настолько особые, чтобы нарушать правила.

Практичность важнее безупречности.

Ошибки никогда не должны замалчиваться.

Если реализацию сложно объяснить — идея плоха.

Если реализацию легко объяснить — идея хороша.

Эти 19 принципов, написанные Тимоти Петерсом в 1999 году, стали философией Python и главной причиной его популярности в обучении. Доступны командой import this в любом интерпретаторе Python.

👑 Великодушный пожизненный диктатор (BDFL)

Гвидо ван Россум руководил развитием Python с 1989 по 2018 год, имея титул «Великодушного пожизненного диктатора» (Benevolent Dictator for Life). Это неформальное звание означало, что последнее слово по всем спорным вопросам оставалось за ним.

В июле 2018 года Гвидо объявил об уходе с поста после споров вокруг PEP 572 (моржовый оператор). Сегодня развитие языка координирует Python Steering Council — совет из 5 человек, избираемых сообществом.

Награды: В 2001 году Гвидо получил награду FSF Award for the Advancement of Free Software. В 2022-м — премию Дейкстры за вклад в информатику.

«Python достаточно быстр для наших нужд, а когда не достаточно быстр — вы пишете модуль на C».

— Гвидо ван Россум

«Я оставляю Python в хороших руках. Сообщество справится без меня».

— Заявление об уходе, 2018

💡 Интересные факты о Python

🎭

Название — не змея

Python назван в честь комедийного шоу «Монти Пайтон», а не в честь змеи. Именно поэтому на логотипе Python — две стилизованные змеиные головы, а не одна.

🎬

Пасхалки в Python

import this — Дзен Python. import antigravity открывает комикс xkcd. from __future__ import braces выводит "not a chance" — шутка про фигурные скобки.

🚀

NASA и Python

NASA использует Python для анализа данных, моделирования и даже для управления марсоходами. Инженеры ценят его за скорость разработки и надёжность.

💼

Кто использует Python

Google (YouTube, Google Drive), Instagram (бэкенд на Django), Netflix (рекомендательная система), Spotify (аналитика), Dropbox (десктопные клиенты).

🐍

Логотип с историей

Изначальный логотип был создан неофициально. Современный логотип с двумя змеями принят в 2006 году и символизирует динамичность языка.

📚

Первый язык в топовых вузах

MIT, Stanford, Harvard, МГУ, ИТМО, ВШЭ используют Python для фундаментального обучения программированию из-за его простоты и выразительности.
